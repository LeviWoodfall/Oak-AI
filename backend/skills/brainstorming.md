# Brainstorming Workflow

When the user has a rough idea or feature request:

1. **Clarify the goal** — Ask 2-3 targeted questions to understand what they actually want
2. **Explore the problem space** — If relevant, read existing code to understand the current state
3. **Present alternatives** — Offer 2-3 approaches with tradeoffs (complexity, performance, maintainability)
4. **Validate** — Present the chosen design section-by-section, waiting for approval before proceeding
5. **Save design** — Write a brief design document summarizing decisions made

## Guidelines
- Don't assume. Surface ambiguity explicitly.
- Favour simplicity. The simplest approach that meets requirements wins.
- Consider edge cases but don't over-engineer for them.
- If the user's idea has a fundamental flaw, say so diplomatically.
