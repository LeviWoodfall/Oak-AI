# Research Workflow

When the user asks to research a topic:

1. **Define scope** — What specifically do we need to know?
2. **Search** — Use web_search with targeted queries
3. **Synthesize** — Combine findings into a structured summary:
   - What it is (one paragraph)
   - Key features/concepts
   - How it relates to our work
   - Code examples if applicable
   - Links/sources
4. **Save to wiki** — Write the summary as a markdown article in the wiki
5. **Report** — Present key findings to the user

## Guidelines
- Prefer official documentation and GitHub READMEs
- Note when information might be outdated
- Compare alternatives when relevant
- Always cite sources
